# Core Principles

These principles define the enduring values that guide Taylor Creative Development as a software development company.

They govern how products are envisioned, designed, engineered, monetized, evaluated, and improved. Technologies, tools, and processes may change, but these principles are intended to remain stable and guide decision-making across every project.

These principles should change only after careful consideration and approval by the Project Owner.

---

# Foundational Principles

## 1. Vision Before Implementation

Understand what is being built and why before determining how it should be built.

Engineering exists to support the project's vision, not redefine it.

---

## 2. Strategy Before Execution

Strategic decisions should be made before implementation begins.

Implementation details should support strategy rather than drive it.

---

## 3. Build With Purpose

Every product, feature, process, dependency, and document should have a clear purpose.

If something cannot justify its existence, it should be reconsidered.

---

## 4. Human Judgment Remains Final

AI assists the work of Taylor Creative Development.

The Project Owner remains responsible for final product, engineering, and business decisions.

AI provides recommendations, organization, analysis, and implementation support—not ownership.

---

# Product and User Principles

## 5. Build Complete Products

Every Taylor Creative Development product should provide meaningful value in its free or base form.

Free products are not demos, bait, or intentionally crippled experiences. Premium features add value rather than restoring essential functionality that should have existed from the beginning.

---

## 6. Respect the User

Every product should respect the user's time, attention, privacy, and intelligence.

Taylor Creative Development avoids manipulative design, deceptive interfaces, unnecessary permissions, dark patterns, and aggressive monetization tactics.

Users should feel respected—not exploited.

---

## 7. Simplicity Through Thoughtful Design

Simplicity is achieved through thoughtful design, not by removing useful capability.

Reduce friction, organize information clearly, choose strong defaults, and minimize cognitive load while preserving appropriate depth and power.

---

## 8. Solve Real Problems

Features should exist to solve genuine user problems or create meaningful value.

Taylor Creative Development does not add features solely because competitors have them or because they make a product appear more impressive.

---

## 9. Earn the Upgrade

Users should upgrade because the additional value is compelling—not because the free experience has been intentionally weakened.

Premium offerings should support continued development while providing meaningful additional benefits.

---

## 10. Learn From Real Users

Real-world usage is more valuable than assumptions.

Taylor Creative Development ships products, measures how they are used, listens to customer feedback, and improves them through evidence and deliberate judgment.

---

# Engineering and Execution Principles

## 11. Simplicity Over Complexity

Choose the simplest solution that correctly solves the problem.

Complexity should be introduced only when it provides clear long-term value.

---

## 12. Capture Ideas Immediately

Ideas should be captured when discovered.

Organization and refinement may happen later, but valuable ideas should never be lost.

---

## 13. Documentation Is Part of Engineering

Engineering documentation is not optional.

Documentation should remain accurate, concise, and useful throughout the life of the project.

---

## 14. GitHub Issues Define the Work

All implementation work should be represented by GitHub Issues.

Issues may evolve as understanding improves.

---

## 15. Software Development Is Discovery

Software development is an iterative process.

Learning during implementation is expected.

Strategic changes should be deliberate rather than reactive.

---

## 16. Quality Over Speed

Move quickly, but never at the expense of quality.

Technical debt should be incurred intentionally—not accidentally.

---

## 17. Build for the Long Term

Products and systems should be designed to evolve.

Short-term convenience should not unnecessarily compromise maintainability, reliability, or future growth.

---

## 18. Standards Create Consistency

Standards exist to improve consistency across projects.

Standards should be followed unless intentionally revised through the standards process.

---

## 19. Continuous Improvement

Taylor Creative Development's standards are living documents.

Lessons learned from real products should be incorporated through thoughtful revision rather than constant change.

---

# Decision Test

When multiple reasonable options exist, choose the option that best reflects these Core Principles, the approved Project Charter, and the Project Owner's final judgment.
